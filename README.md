I study Creative Tech & AI at Howest and spend most of my time turning rough AI ideas into working software. The pattern is pretty simple: understand the model well enough, wrap it in a clean interface, then ship something a real person can use.

Recent work includes shipping AI software for the Gates Foundation, a semester abroad at George Mason, and building AIGENIZE as a two-week custom software sprint studio.

## Worth opening first

- [TrackMeta](https://github.com/TobiasPottier/TrackMeta) — a macOS menu bar app for Claude Max usage tracking and session control.
- [LMR — Live Movie Recommendation](https://github.com/TobiasPottier/LMR---Live-Movie-Recommendation) — a full-stack movie recommender using content-based filtering, MovieLens, TMDB, MongoDB, and a web UI.
- [Custom Neural Network Learns MNIST](https://github.com/TobiasPottier/Custom-Neural-Network-Learns-MNIST) — a neural network written from scratch, then hooked up to live drawing inference.
- [AI Plays Tetris](https://github.com/TobiasPottier/AI-Plays-Tetris) and [Flappy Bird — The Reinforced Way](https://github.com/TobiasPottier/Flappy-Bird---The-Reinforced-Way) — small reinforcement learning experiments with games, rewards, and plenty of imperfect runs.
- [Post Bariatric Surgery Prediction](https://github.com/TobiasPottier/Post-Bariatric-Surgery-Prediction) — ML work around post-surgery complication risk in a clinical context.

## What I like building

- Developer tools I would actually keep open.
- AI products where the model is only one part of the experience.
- Automation that saves time without making the user give up control.
- Small, shippable systems instead of endless prototypes.

## Tools I reach for

Python, Swift, JavaScript, C#, PyTorch, scikit-learn, MongoDB, Docker, GitHub Actions, and whatever boring tool gets the idea into someone's hands faster.

## If you're here to get a read on me

I use AI heavily, but I do not want my work to feel like autocomplete pasted into a repo. I care about taste, clear constraints, and knowing enough about the system to defend the choices in it.
